# Игра про ценности компании

## Описание

MVP веб-игры, в которой пользователь проходит игровой путь, знакомясь с ключевыми ценностями команды: ответственность, прозрачность и скорость.

## Стек

- [Next.js](https://nextjs.org/) (App Router)
- TypeScript
- Tailwind CSS
- [React Three Fiber](https://docs.pmnd.rs/react-three-fiber/getting-started/introduction) + [Drei](https://github.com/pmndrs/drei) + [Three.js](https://threejs.org/) — 3D-сцена комнаты и персонажа
- Feature-Sliced Design (FSD) — слои и импорты в `src/`

## Установка и запуск

### Требования

- **Node.js** 20+ (рекомендуется LTS)

### Установка зависимостей

```bash
npm install
```

### Режим разработки

```bash
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000) в браузере.

### Сборка и продакшен

```bash
npm run build
npm run start
```

### Линтинг

```bash
npm run lint
```

## Структура проекта и FSD

Проект следует идеям [Feature-Sliced Design](https://feature-sliced.design/): слайсы по доменам, однонаправленные зависимости сверху вниз (от приложения к общим модулям).

### Слои в `src/`

| Слой | Папка | Назначение |
|------|--------|------------|
| **App** | `app/` | Маршруты Next.js (App Router), корневой `layout`, глобальные стили. Страницы подключают фичи и сущности, не содержат тяжёлой бизнес-логики. |
| **Entities** | `entities/` | Бизнес-сущности: типы и данные (например, миссии в `entities/mission/`). Без UI-сцен. |
| **Features** | `features/` | Законченные возможности продукта: 3D-комната и игрок (`game-scene/`), оверлеи миссий (`mission-overlay/`), карточки (`mission-card/`), эпичный оверлей (`ui-epic-overlay/`). |
| **Shared** | `shared/` | Переиспользуемый UI и утилиты без привязки к домену (например, `shared/ui/`). |

В репозитории **нет** отдельных папок `pages/` и `processes/` — маршруты живут в `app/`, игровые сценарии разнесены по фичам (`game-scene`, `mission-overlay`).

### Правила импортов (упрощённо)

- `app` → может импортировать из `entities`, `features`, `shared` и собирать экран из нескольких фич.
- `features` → в основном `entities` и `shared`; перекрёстные импорты между разными фичами по возможности избегаются — общее выносится в `entities` или `shared`, композиция страницы остаётся в `app/`.
- `entities` → `shared` и внутренние типы/данные слоя.
- `shared` → не зависит от доменных слоёв.

Так проще сопровождать код и подключать новые миссии без лишних связей между модулями.

## Миссии

1. Ответственность
2. Прозрачность
3. Скорость

## Что реализовано сейчас

- Лендинг и переход в игровую сцену.
- 3D-комната с персонажем, NPC, HUD миссий и оверлеями под каждую ценность.
- Тач-управление на узких экранах, достижения и контекст кампании через React Context (`GameContext` в `features/game-scene`).

## Деплой

Проект можно задеплоить на [Vercel](https://vercel.com/) через стандартный процесс деплоя Next.js (`npm run build`).
